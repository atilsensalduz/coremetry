{{- if and (gt (int .Values.replicaCount) 1) (eq .Values.service.sessionAffinity "None") }}

⚠ MCP + SSE session affinity is disabled but you're running multiple
  replicas. MCP sessions and SSE streams hold pod-local state — a
  reconnect that lands on a different pod will lose the session. Two
  fixes available:
    1. Set service.sessionAffinity=ClientIP (default) for k8s-native
       per-source-IP stickiness (fine for direct in-cluster clients).
    2. Front the Service with an ingress that injects a sticky cookie
       — example for nginx-ingress:
         metadata.annotations:
           nginx.ingress.kubernetes.io/affinity: "cookie"
           nginx.ingress.kubernetes.io/session-cookie-name: "coremetry-route"
           nginx.ingress.kubernetes.io/session-cookie-max-age: "10800"
       Use option 2 when clients are external (NAT collapses ClientIP).
{{- end }}
